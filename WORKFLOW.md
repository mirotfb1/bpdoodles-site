## First-time Setup (Cloudflare Pages + GitHub)

1. **Create GitHub repo**
   ```bash
   cd C:\MacroNews\bpdoodles-site
   git init
   git add .
   git commit -m "Initial site: homepage, cockpit, W12 newsletter, calendar placeholder"
   # Create repo on GitHub (e.g. bpdoodles/bpdoodles-site), then:
   git remote add origin https://github.com/bpdoodles/bpdoodles-site.git
   git branch -M main
   git push -u origin main
   ```

2. **Connect to Cloudflare Pages**
   - Cloudflare dashboard → Pages → Create a project → Connect to Git
   - Select the `bpdoodles-site` repo
   - Build settings: **Build command:** *(leave empty)* · **Output directory:** `/`
   - Deploy

3. **Attach custom domain**
   - Pages project → Custom domains → Add `bpdoodles.com`
   - Since the domain is already on Cloudflare DNS, it will auto-configure (CNAME or direct)
   - Also add `www.bpdoodles.com` → redirect to apex

4. **Set up Formspree** (email waitlist on homepage)
   - Sign up at https://formspree.io (free tier: 50 submissions/month)
   - Create a new form → copy the form ID (looks like `xabcdefg`)
   - In `index.html`, replace `YOUR_FORM_ID` with your actual ID:
     ```html
     <form ... action="https://formspree.io/f/xabcdefg" ...>
     ```
   - Submissions arrive at your email. Export CSV when needed.

## Weekly Publishing Workflow

Run every Monday/Tuesday after finalizing the newsletter.

### Step 1 — Finalize newsletter
Newsletter draft lives at: `C:\MacroNews\data\newsletters\drafts\YYYY-WNN-v3.html`
(See `data/newsletters/WORKFLOW.md` for the full editorial process)

### Step 2 — Copy newsletter to site
Do this **after** all patch scripts have run (badges are baked in during patching).

```bash
WEEK="2026-W13"
mkdir -p bpdoodles-site/newsletters/$WEEK
cp data/newsletters/drafts/${WEEK}-v3.html bpdoodles-site/newsletters/$WEEK/index.html
```

⚠️ Always copy last — the patch scripts write to `data/newsletters/drafts/`, not to the site folder.

### Step 3 — Update newsletter archive
Open `bpdoodles-site/newsletters/index.html` and add a new `<a class="issue-row">` block
**at the top** of the issue list (newest first). Copy the existing W12 block and update:
- `href` → `/newsletters/YYYY-WNN/`
- `issue-row-meta` → week number + date (EN + BG)
- `issue-row-title` → theme of the issue (EN + BG)
- `issue-row-desc` → one-line summary (EN + BG)

### Step 4 — Update homepage "Latest Issue" card
Open `bpdoodles-site/index.html` and update the `.issue-card` block:
- `issue-meta` → new week + date
- `issue-title` → new theme
- `issue-desc` → one-line summary
- `issue-link href` → `/newsletters/YYYY-WNN/`

### Step 5 — Refresh cockpit
```bash
cd C:\MacroNews
python data/gen_cockpit_site.py
```
This regenerates `bpdoodles-site/cockpit/charts/mXX.png` and `cockpit/manifest.json`
with the latest data. Run this after fetching fresh data.

### Step 6 — Commit and push
```bash
cd C:\MacroNews\bpdoodles-site
git add .
git commit -m "Publish YYYY-WNN · [theme one-liner]"
git push
```
Cloudflare Pages auto-deploys in ~30 seconds. Done.

## Cockpit Update Workflow

### Refresh existing charts (weekly)
```bash
python data/gen_cockpit_site.py
```
Outputs to `bpdoodles-site/cockpit/charts/` and `cockpit/manifest.json`.
Then commit and push as above.

### Add a new chart to the cockpit

1. **Write the chart generator** — follow the pattern in `data/gen_cockpit_m01.py` (or similar).
   Output: a PNG saved to `bpdoodles-site/cockpit/charts/mXX.png`

2. **Register in gen_cockpit_site.py** — add the new chart's metadata to the `CHARTS` list
   (code, title EN/BG, subtitle, dataset, footer text, section).

3. **Run gen_cockpit_site.py** — this regenerates the manifest and all chart PNGs.

4. **Update manifest.json section groupings** if the new chart belongs to a new section.

5. **Commit and push** — Cloudflare deploys automatically.

The cockpit `index.html` does **not** need to be modified when adding charts — it reads
everything dynamically from `manifest.json` at load time.

**Current chart inventory:** M-01 to M-26 (26 charts). M-27 to M-32 exist in the
internal dashboard but are excluded from the public cockpit site for now.

## Adding a New Site Page

1. Create `bpdoodles-site/PAGENAME/index.html`
2. Copy the top-bar nav + masthead + footer from any existing page
3. Update the `active` class on the nav link for the new page
4. Add a link to it in the nav strip of **all other pages** (index, cockpit, newsletters, calendar)
5. Commit and push
